# Traceability Best-Effort Note

Traceability in this workflow should be treated as **best-effort audit support**, not proof of perfect completeness.

## What traceability is good for

- showing where a canonical principle came from
- making it easier to challenge or update a principle
- revealing over-compression and missing caveats
- helping future synthesis work stay grounded

## What traceability cannot honestly guarantee

- that every sentence in every source was represented perfectly
- that no inference entered the synthesis
- that all merged items are lossless in a strict formal sense

## Recommended standard

For each canonical principle, provide:

- source file(s)
- section name or recognizable anchor
- short note on whether the principle is:
  - directly stated
  - synthesized from several sources
  - inferred from repeated patterns

If an item is weakly supported, say so.
If an item is mostly a design decision layered on top of the corpus, say so.

Honest traceability is much more useful than performative certainty.
