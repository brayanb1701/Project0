# Headless Mission Context Template

Use this as the compact bigger-picture prefix before the scoped task for a child headless session or subagent.

```md
## Mission context
We are working on `<initiative-or-project-goal>`.
This task exists to `<why-this-workstream-exists>`.
Its output will be consumed by `<downstream-user-artifact-or-decision>`.
Respect these bigger-picture constraints: `<constraints>`.

## Your task
<scoped-task>

## Deliverable
<expected-output>
```

Keep the mission context short. It should orient the child, not duplicate the whole parent prompt.
