# Headless Launch Response Template

Use this when reporting a launched headless run back to the user.

```md
Launched `<harness>` headless session.

- Session name: `<session_name>`
- PID: `<pid>`
- Project code: `<project_code>`
- JSONL log: `<jsonl_path>`
- stderr sidecar: `<stderr_path>`
```

Keep the wording compact. The critical fields are the session name, PID, and JSONL path.
